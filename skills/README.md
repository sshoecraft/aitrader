# Skills — prose playbooks for the agent

Skills are **guidance you read and apply with judgment**, not scripts and not
rules. They are how an experienced colleague would describe *how to think* about
a recurring situation — never *what to decide*. They contain no thresholds, no
scores, no entry/exit triggers, no sizing formulas. If a skill ever hardens into
"if X then do Y," it has become strategy-in-prose and betrays the mandate; rewrite
it as judgment.

The agent is free to deviate from any skill when the situation warrants — and
should note in the journal when and why it did.

These are seeds. The right way to improve the trader is to refine this prose and
add skills as you learn — **never** by adding decision code (BRIEF §8).

- `exit-thinking.md` — how to think about getting out.
- `journaling-and-idempotency.md` — memory discipline + never double-submitting.
